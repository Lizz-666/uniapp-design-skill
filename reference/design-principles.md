# Mobile Design Principles for Mini-Programs

借鉴专业设计原则，适配移动端和小程序平台的约束。

## 排版

### 核心原则

- 移动端正文字号 **28-30rpx**，比 Web 端（16px）略大
- 标题用 **2-3 级**层次，不需要 Web 端的 5-6 级
- 行高 **1.5-1.8**，比 Web 略宽松
- 每行 **15-20 个汉字**为宜，最多不超过 25 个
- 同一页面不超过 **2 种字重**（400 + 600 足够）

### 层次对比

```
页面主标题:  48rpx / 600 weight
区块标题:    36rpx / 600 weight
正文:        28rpx / 400 weight / 1.6 line-height
辅助文字:    24rpx / 400 weight / color: gray-500
```

### 避免

- 全大写英文标题（移动端难以阅读）
- 过细字重（300 以下在小屏幕上发虚）
- 大段居中文字（超过 3 行应左对齐）
- 单行文字用 rpx 行高，多行用 1.5+ 的 line-height

## 色彩

### 移动端特殊考虑

- **户外可读**：手机可能在阳光下使用，对比度要比 Web 更高
- **WCAG AA 最低要求**：正文与背景对比度 ≥ 4.5:1
- **避免纯黑文字**：用 `#1a2332` 替代 `#000000`，减轻视觉疲劳
- **避免纯白背景 + 纯灰文字**：背景用 `#fafafa`，文字用带色调的灰

### 色彩比例

```
60% — 中性色（背景、卡片、分隔线）
30% — 次要色（正文、副标题、边框）
10% — 强调色（CTA 按钮、关键图标、未读标记）
```

移动端强调色可以比 Web 端占比略高，因为屏幕小，需要更明确的视觉引导。

### 渐变

小程序支持 `linear-gradient`，但：

- 小面积使用（按钮、标签），不要大面积背景渐变
- 渐变方向保持简单：`to right` 或 `to bottom`
- 颜色变化跨度不要太大（同色系深浅变化优于跨色相）

## 空间与布局

### 移动端间距原则

- 卡片内间距：**24-32rpx**
- 列表项高度：**≥ 88rpx**（触摸友好）
- 区块间距：**32-48rpx**
- 页面两侧留白：**24-32rpx**
- 底部安全区：`env(safe-area-inset-bottom)`

### 触摸友好

- 最小触摸目标：**88 × 88 rpx**
- 按钮间距：**≥ 16rpx**
- 列表项操作区域不要放在最右边（右手拇指难以触达）
- 避免过小的可点击区域（如图标 < 48rpx 又没有 padding）

### 卡片设计

- 一行放 **1-2 个卡片**，不要 3 个以上
- 卡片圆角 **12-16rpx**（太大会在多卡片布局中显得拥挤）
- 卡片阴影尽量轻：`0 4rpx 16rpx rgba(0,0,0,0.08)`
- 不要卡片套卡片

### 信息密度

- 一屏展示 **3-5 条**核心信息
- 用折叠/展开处理次要内容
- Tab 切换优于长页面滚动
- 数字数据放大展示，说明文字缩小

## 导航模式

### TabBar（底部导航）

- 3-5 个 Tab 为宜，不超过 5 个
- Tab 图标 + 文字，不要只有图标
- 当前 Tab 用主色高亮，其他用灰色
- 图标风格统一（线性/填充 不要混用）

### 层级导航

- 3 层以内最佳，超过 4 层用户会迷路
- 用面包屑不现实，靠返回按钮 + 标题
- 页面标题要明确，让用户知道在哪

### 临时导航

- 弹窗/ActionSheet/Picker 属于临时导航
- 不要在弹窗里再弹弹窗
- 弹窗内容尽量简单，复杂内容用新页面

## 表单设计

### 布局

- Label 在上，Input 在下（移动端标准）
- 或 Label 在左、Input 在右（设置页面常用）
- 输入框高度 ≥ **80rpx**
- 表单项之间间距 **24-32rpx**
- 提交按钮放在页面底部或表单最下方

### 交互

- 输入时实时校验，不要等提交后
- 错误提示在输入框下方紧邻位置
- 用 `<picker>` 替代手动输入（日期、地区、选择列表）
- 必填项用 `*` 或颜色标记
- 长表单分步骤，每步 3-5 个字段

## 动效

### 小程序中的动画

优先用 CSS `transition`：

```css
.fade-enter {
  opacity: 0;
  transform: translateY(20rpx);
}
.fade-enter-active {
  opacity: 1;
  transform: translateY(0);
  transition: all 0.25s ease-out;
}
```

### 原则

- 时长 **200-350ms**，不超过 500ms
- 缓动用 **ease-out**（cubic-bezier(0.22, 1, 0.36, 1)）
- **不用 bounce/elastic**：卡顿且不专业
- 一次只做一个动画，不要多个元素同时动
- 列表 stagger 效果用 `transition-delay` 实现：

```css
.list-item:nth-child(1) { transition-delay: 0ms; }
.list-item:nth-child(2) { transition-delay: 50ms; }
.list-item:nth-child(3) { transition-delay: 100ms; }
```

### 适合小程序的动效

- 页面切换：滑动（系统自带）
- 列表加载：fade in + translateY
- 按钮：scale 微缩 + 色变
- 展开/收起：height 过渡（用 max-height hack）
- 数值变化：数字滚动（简单 JS 定时器）

## 加载与空状态

### 加载

- 骨架屏优于 loading 转圈
- 骨架屏用灰色块占位，形状对应真实内容
- 快速操作（< 1s）不需要 loading
- 网络请求必须 loading + 超时提示

### 空状态

- 不要只显示"暂无数据"
- 提供引导操作（"去添加"、"去逛逛"）
- 配插图增加亲和力
- 区分：无内容 / 无搜索结果 / 网络错误

### 下拉刷新

```json
// pages.json
{ "enablePullDownRefresh": true }
```

配合 `onPullDownRefresh` + `uni.stopPullDownRefresh()` 使用。

## 列表设计

- 列表项高度统一，不要参差不齐
- 列表项左对齐，右侧放次要信息或操作
- 用分隔线或卡片间距区分列表项
- 长列表底部显示"加载更多"或"到底了"
- 支持下拉刷新 + 触底加载更多
